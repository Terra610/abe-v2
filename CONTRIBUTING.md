# Contributing to A.B.E.

Thank you for your interest in strengthening constitutional fidelity.

## Core Rules
1. No server code. Ever.  
2. No data collection.  
3. No monetization.  
4. All modules must be deterministic and transparent.  
5. No dependencies that phone home.  
6. All contributions must pass divergence integrity checks.  

## How to Contribute
1. Fork the repository  
2. Create a feature branch  
3. Add tests for new modules  
4. Submit a pull request with:
   - purpose of change  
   - constitutional theory reference  
   - expected impact  

## Security
If you discover an issue that compromises user privacy or constitutional accuracy, email the project lead immediately.
